# SL P&L計算バグ修正 実装計画

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** STOP_LOSS_TRIGGEREDイベントの`error`フィールドから`unrealized_pnl`を抽出し、SL trip の正しいP&Lを計算する

**Architecture:** `data_loader.py` の `_calc_trip_fields` を変更。SL trip の場合のみ、`(close_price - open_price) * dir` ではなく `error` フィールドからparseしたunrealized_pnlを使用。新しいヘルパー `_parse_sl_pnl` を追加。

**Tech Stack:** Python, 既存backtester

## 背景

`STOP_LOSS_TRIGGERED` イベントの `price` フィールドは実際の約定価格ではなく、別の値（mid_priceとも一致しない）が記録されている。一方 `error` フィールドには `unrealized_pnl=X` の形で実損失が記録されている。

現状: `pnl = (close_price - open_price) * size * dir` で計算 → SL trip P&Lが+方向に大きく誤計算
影響: 4/2の場合、SL補正で+703 JPYの過大計算 → 5日合計が+1481(誤) vs -508(実)

```python
def _parse_sl_pnl(error_str: str) -> float:
    """STOP_LOSS_TRIGGERED の error フィールドから unrealized_pnl を抽出。

    Rust側のSL記録は error="unrealized_pnl=-17.469" のような形式。
    SLイベントの price フィールドは実約定価格ではないため、
    error フィールドから真のP&Lを取得する必要がある。

    Args:
        error_str: TradeEvent.error の値

    Returns:
        unrealized_pnl の値 (JPY)。パース失敗時は 0.0。
    """
    if not error_str or "unrealized_pnl=" not in error_str:
        return 0.0
    try:
        # "unrealized_pnl=" の後の数値を抽出
        after = error_str.split("unrealized_pnl=", 1)[1]
        # 数字以外で区切る (空白、カンマ、改行など)
        token = ""
        for ch in after:
            if ch.isdigit() or ch in "+-.":
                token += ch
            else:
                break
        return float(token) if token else 0.0
    except (ValueError, IndexError):
        return 0.0
```

(b) `_calc_trip_fields` の `pnl_jpy` 計算を変更:

現状:
```python
    # pnl_jpy = (close_price - open_price) * size * direction
    pnl_jpy = (close_fill.price - open_fill.price) * size * direction
```

変更後:
```python
    # SL trip: priceフィールドが実約定価格でないため、
    # errorフィールドのunrealized_pnlから実損失を取得
    if sl_triggered:
        pnl_jpy = _parse_sl_pnl(close_fill.error)
    else:
        pnl_jpy = (close_fill.price - open_fill.price) * size * direction
```
